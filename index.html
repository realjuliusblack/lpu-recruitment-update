The Hierarchy

You sit above the stack. Four tiers do the work.

LPU runs on a four-tier hierarchy borrowed from how real companies work. You are the principal. You never enter the stack. You give one directive to The Top Bot, the Operator tier, and it routes everything down to the Recruitment Manager and its six agents, then reports everything back up. Each tier only talks to the tiers directly above and below it.

Tier 1
Operator
The Top Bot. Your single interface across every Locums brand. Takes your directive, routes it down the stack, reports results back up. Never touches execution itself.
Tier 2
Manager
Recruitment Manager. One department head. Translates The Top Bot's directive into a plan. Delegates to the six recruitment agents. Reports back up.
Tier 3
Agent
Six domain experts. Job Sourcing, Physician Finder, Qualification, Name Clear, Background, Facility Presentation. Each decides what to do and when.
Tier 4
Engine
The thing that actually does the work. Ingests a vendor feed. Writes a JD. Runs a background check. No autonomy, just execution. Called by an Agent.
Why this matters
You never manage agents or debug engines. You message The Top Bot, work flows down the stack, and the answer comes back to you in the same thread. One directive in, one answer out.

What a Managed Agent is
Every agent runs AI reasoning on top of deterministic tools. The AI decides what to do and when. The tools execute with no ambiguity. Hard rules in code, judgment calls in the agent. This is why agents self-heal, problem-solve, and adapt without breaking production workflows.

Agent 2 · Physician Finder · In Build

Agent 2 knows who to call first.

Agent 2 turns a live job into the right physician in your GHL. Built as a Managed Agent: AI reasoning on top, deterministic LPU tools underneath. Phase A build opens this week.

Step 01 · Trigger
A new job surfaces
Agent 1 publishes a row. Agent 2 sees it immediately and starts the match run against your existing GHL list.
Step 02 · Match
Ranked, not filtered
Specialty, sub-specialty, licensure state, rate expectation, availability window, prior responsiveness. Every candidate gets a score.
Step 03 · Reach
Personalized outreach
One email per match. LPU voice. Rate, facility, shift pattern stated plainly. Not a blast. Not a template. Each one written for the candidate.
Step 04 · Reply
Read, classify, route
When a candidate responds, Agent 2 reads the reply, classifies intent, and routes. Interested goes to Agent 3. Unsubscribes are honored instantly.
Step 05 · Loop
Never spam, always warm
Non-responders get re-touched only when the next genuinely matching job lands. No weekly newsletters, no drip sludge, no burned list.
Step 06 · Report
Up to The Top Bot
Every match, every send, every reply logged. The Top Bot reports cadence, conversion, and candidate quality back to you on request.
Why a Managed Agent
The reasoning layer handles the judgment calls: is this match worth emailing, which tone fits this candidate, when to follow up. The tool layer stays deterministic: GHL writes, match scores, email sends, unsubscribe handling. Thin agent, thick tools. Same architecture every LPU agent uses.

The Method · Black Label Dev Cycle

Four phases. No shortcut. That is why Agent 1 shipped clean.

Every engine inside every agent moves through the same four phases. One engine at a time. No engine leaves Build until Build passes. No engine leaves Iterate until it runs clean against real dependencies. No engine goes live until Certify returns green. This is why the 8 out of 8 on Agent 1 is real.

Phase 01
Plan
Agenda locked. Contract written. Inputs, outputs, dependencies, known failure modes, test criteria. All on paper before any code.
Phase 02
Build
Implement to the plan. Save locally first. Deploy after. No scope creep. No surprise features. The plan is the fence.
Phase 03
Iterate
Run against real dependencies. Real LocumSmart, real Aya, real GHL, real Claude. No mocks. Every run logged with a pass or fail and a fix.
Phase 04
Certify
Automated cert spec runs. Three verdicts: CERTIFIED, CONDITIONAL, REVOKED. Only CERTIFIED ships. Everything else goes back to Iterate.
Cert is not forever
Cert re-runs on every code change. Cert re-runs on every dependency change. Cert auto-expires after 14 days. An agent is not trusted because it once passed. It is trusted because it is currently certified. This is the only reason LPU can ship production software one engine at a time without breaking.
